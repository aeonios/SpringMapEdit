Requirements

Supported Platforms: all which support Java, SWT, and JOGL, this includes, but is not limited to:

  • Windows
  • Linux
  • OSX (needs testing. someone could supply some startup script?)
Hardware:
  • OpenGL 2.0 capable Graphics Card (Geforce5 and up)
  • Plenty of RAM (depending on mapsize)
Software:
    Java (I recommend JRE 1.7 OpenJDK - ​http://www.java.com) SWT 4.5 library (included in this package - ​http://www.eclipse.org/swt/ ) JOGL 1.1.1 library (included in this package - ​https://jogl.dev.java.net/ )

Start SpringMapEdit

Installation Just extract contents of this archive into any directory. Then start the appropriate script. Windows: one of the "start_win32/64.bat" files. Linux: one of the "start_linux_x86/64.sh" files. OSX: someone needs to supply me those files. (Basically it should be the same commandline as for windows/linux) If you get crashes when editing large maps, or it does not start at all, try editing the startup file. There is a variable which reads like: maxmem=1024m The 1024m is the maximum amount of RAM(in megabytes, note the m) which Java is allowed to use. Set it to your needs, but make it a multiple of 128. If Java can not find a batch of RAM as large as given number, it won't start, so just set it a little lower than your real RAM. If it does not startup properly, you may need to get the correct libraries for your platform, and check some dependencies SWT and JOGL have. (Especially on OSX)

Generating a random heightmap

Click on Edit->Randomize Terrain it will randomize the heightmap, if Random Seed is left at -1.
If you specify some other Random Seed it will create the same terrain over and over again when the setting is not changed.
Height and area granularity can be set with the respective controlbars.

For manual editing select Height from the brush modes and paint on the map.
This dialog lets you choose a brush to edit with Select Brush from the editing mode dialog.

If you want another brush, click Brushmode->Select Brush and choose the one you want. Size is changeable by setting Brush Size

Heightmodes explained:

ModeDescription
RaiseUses selected brush to raise (lower with RMB) the heightmap. Brush Strength determines the Speed.
SmoothSmoothes heightmap inside brush area, ignoring selected brush. Brush Strength determines the maximum height which will be smoothed. Low values preserve cliffs.
SetSets heightmap to Brush Strength value.
ErodeLMB applies Wet Erosion Brush Strength times to brush area, ignoring selected brush. RMB applies Dry Erosion.
RandomizeRandomly adds/subtracts Brush Strength from heightmap inside brush area, ignoring selected brush.

Texturing your heightmap

To automatically paint your map click TexGen->TexGen Whole map this takes its time, but produces a fairly good start.

For manual editing you select Texture from the brush modes and click Texturemode->Select Texture.
This dialog lets you choose a texture to paint with.

Texturemode->Select Color lets you pick a single-colored texture. This is useful for darkening/brightening/coloring the map if used with Texturemode->Blend with low Brush Strength

Texturemodes explained:

ModeDescription
SetThis mode just fills the whole brush area with selected texture, ignoring the brush. Brush Strength determines the Brightness.
AddNo idea how this could be useful :)
BlendDefault mode. Just does what you would expect. Brush Strength determines the Opaqueness.
StampStamps selected texture onto map. Respects alpha channel of texture and ignores brush. Useful for splatting metalpatches.
TexGenUses TexGen-settings to paint inside brush area. Useful for repairing small parts of the map, while preserving the rest.

Adding Features to your map

Select Feature From the brush editing modes.
Pick a feature from the Select Feature Button in the editing mode dialog and place it where you like on the map.

Saving your map

Click on File->Save If you have made a new map it will prompt you where to save the first time, after that it will overwrite that file with each subsequent save.
If you want to save your current map as another file click Save As and specify where to save it, all subsequent Save commands will use that file.

Opening other maps

It is not advised to open maps by that were not made with spring map edit as they may reference features that are not available to it.
However if you wish to do so anyway you can simply open the sdz or sd7 with a program such as p7z or similar and open the smf file like a regular map you made yourself.
Not that features the map has that sme does not will not be usable or maybe even lost when saving again, you can try to import those features manually but it is not advised to do so.

Exporting your map to spring

To export your map, first copy the included folder called template_map.sdd into your spring maps folder.
Save your map as 'map' in that folder and edit mapinfo.lua with a generic text editor, change the name and author fields (make sure they are surrounded by " on both sides and have a , after that).
Your map should then be visible under that name in (spring)lobby, make sure to update your cache if you import it while the lobby is running (Tools->Reload Games/Maps in springlobby).
To share your map with others simply zip or 7z compress the contents inside the template folder (not the folder itself) and upload it to Springfiles.
